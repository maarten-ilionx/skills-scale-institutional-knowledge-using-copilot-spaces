# OctoAcme Project Management Docs

Centralized hub for all OctoAcme project management processes, checklists, and key templates.

## Overview

OctoAcme delivers projects using a repeatable, lightweight framework built around five lifecycle stages: **Initiation → Planning → Execution → Release → Close & Retrospective**. Each stage has clear entry/exit criteria and produces specific artifacts—such as the Project One-pager, Risk Register, sprint backlog, and release checklist—so teams always know what "done" looks like before moving forward.

Roles are intentionally defined to minimize ambiguity. The **Project Manager (PM)** owns coordination, schedule, risks, and communications. The **Product Manager (PdM)** defines outcomes and prioritizes the backlog. **Developers** implement and collaborate on design and testability. **QA/Testing** validates acceptance criteria and quality. **Stakeholders** provide inputs and approvals. Clear role ownership keeps decisions fast and reduces hand-off friction across the lifecycle.

Communication follows a structured cadence: twice-weekly standups for the delivery team, weekly PM–PdM syncs, monthly stakeholder updates, and ad-hoc escalations when blockers surface. Status updates use consistent templates (progress, next steps, risks/blockers, asks/decisions) so the project board and shared docs remain the single source of truth. Escalation paths are explicit—from team triage to PM-led escalation to product leadership to sponsor-level when business impact warrants it.

Quality assurance is built into both daily development and the release gate. Teams favor small PRs linked to issues and acceptance criteria, with CI checks (tests, lint, security scanning) and at least one required approval before merge. Testing expectations cover unit, integration, and end-to-end smoke tests for critical flows. Releases follow a standardized checklist—CI green, release notes and rollback plan ready, staging validation complete—followed by post-deploy verification, stakeholder announcements, and a retrospective that converts learnings into owned, time-bound action items.
